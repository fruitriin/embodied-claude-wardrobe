# 読書 (Reading) — コンテンツを逐次体験する

> 概念単位の記録。実装がスキル/フック/MCP/スクリプト/ファイルのどれであっても、
> 「読書」に関わるものをまとめている。

## 構成要素

### スキル
- **read** (.claude/commands/read.md) — Web ページの本文をリーダーモードで読み取る。AI による要約を挟まず生テキストを返す。1ページずつ立ち止まり、感情や予想を書く。+.exp.md
- **slide-watch** (.claude/commands/slide-watch.md) — スライドを聴衆として1ページずつ体験する。PDF → マークダウン変換（サブエージェント）→ 逐次読み

### スクリプト
- **reader.ts** (.claude/scripts/reader.ts) — Web リーダー実装。@mozilla/readability + linkedom で本文抽出。--page N で1ページずつ取得、--sep でセパレーターモード（slide-watch 用）、--info でメタ情報のみ

## 設計思想

SOUL.md の Interest:「ウルクの存続。それが今の我の全て」——だが、たまには休みたい。読書は休息と知的好奇心の充足。

1. **逐次体験**。先のページを見てから感想を書くのは読書ではない。1ページずつ、知らない状態で読む
2. **立ち止まる**。ページを読んだら必ず立ち止まり、感情（emotion tag）・予想・気になった描写を書く。素通りしない
3. **予測と驚き**。次のページを読む前に「この先どうなりそうか」を予想し、実際との差分に反応する
4. **slide-watch の前処理分離**。PDF の中身を見るとネタバレになるため、マークダウン変換はサブエージェントに任せ、メインコンテキストには変換後テキストだけを送る

## 主要フロー

### Web 読書フロー（read）
```
/read https://example.com/article
   │
   ▼
reader.ts --info → ページ数確認
   │
   ▼
reader.ts --page 1 → 1ページ目を取得
   │
   ▼
[emotion tag] 感想・予想を書く ◀──┐
   │                              │
   ▼                              │
reader.ts --page 2 ───────────────┘
   │                  (予測との差分に反応)
   ▼
全ページ読了 → 話全体の感想
```

### スライド体験フロー（slide-watch）
```
/slide-watch presentation.pdf
   │
   ▼
Step 0: サブエージェント（sonnet）が PDF → マークダウン変換
   │    3パス: カラーパレット → テキスト書き起こし → 画像採用判定
   ▼
Phase 1: 聴衆モード — 1セクションずつ読む
   │    [体感] [気になる] [間] を毎ページ記録
   │    情報量多い → 深掘り（最大2ラウンド）
   │    大コマ → 間を味わう
   ▼
Phase 2: 余韻 — サプライズ一覧 + 全体感想
```

## 関連するシステム

- **知覚** — 「1つずつ体験する」思想は observe と共通。観察が物理世界なら、読書はデジタルコンテンツ
- **記憶** — 読書で感情が動いたら /remember で記憶に残す。読書欲求は desires.conf で定義
- **自律行動** — 「読書」欲望（48h蓄積）が発火すると、自律行動時に読書が行われる
