# 計画: アップストリームコントリビューションフロー

> ダウンストリームでの改善をアップストリーム（embodied-claude-wardrobe）に還元するための仕組み。

## 背景

ダウンストリームで行われた改善がアップストリームに還元されないまま埋もれる問題。
PR は一般ユーザーには重い。代わりに **Issue ベース** で敷居を下げる。

## 設計

### フロー概要

```
セッション中の改善
  ↓
セッション終了時: wd-contribution が差分スキャン
  ↓ フィードバックがあれば
.claude/upstream-backlog.jsonl に追記
  ↓ セッションカウンタをインクリメント
N セッション溜まったら Issue 送信を提案
  ↓ ユーザーが承認
gh issue create で upstream に Issue を送る
```

### コンポーネント

#### 1. セッションカウンタ

**ファイル**: `.claude/upstream-session-counter` (永続。/tmp ではなくプロジェクト配下)

既存の hearing-hook / continue-check と同じパターン:
- テキストファイルに整数1つ
- セッション終了時にインクリメント（フィードバックがあった場合のみ）
- Issue 送信後にリセット

```
# 読み取り
COUNT=$(cat .claude/upstream-session-counter 2>/dev/null || echo 0)
# インクリメント
echo $((COUNT + 1)) > .claude/upstream-session-counter
# リセット
rm -f .claude/upstream-session-counter
```

**閾値**: `UPSTREAM_ISSUE_THRESHOLD=5`（デフォルト。5セッション分溜まったら提案）

#### 2. バックログファイル

**ファイル**: `.claude/upstream-backlog.jsonl`

JSONL 形式。1行1エントリ:

```jsonl
{"date":"2026-03-29","session":"abc123","type":"bugfix","target":"memory-mcp/store.py","summary":"埋め込み次元数ミスマッチ検出","intent":"undecided"}
{"date":"2026-03-29","session":"abc123","type":"improvement","target":"CLAUDE.md","summary":"wd- プレフィックス統一","intent":"issue"}
```

フィールド:
- `date` — 検出日
- `session` — セッション識別子（コミットハッシュ等）
- `type` — `bugfix` / `improvement` / `feature` / `refactor`
- `target` — 対象ファイル or モジュール
- `summary` — 1行要約
- `intent` — `issue` / `local` / `undecided`（ユーザーがタグ付け。デフォルト `undecided`）

#### 3. シャットダウン手順への統合

BOOT_SHUTDOWN.md のシャットダウン手順に追加:

```
5. アップストリーム還元チェック:
   - 今回のセッションで変更されたアップストリーム管理対象ファイルがあれば
   - wd-contribution の軽量版（git diff のみ、docs/project-overview は読まない）を実行
   - フィードバックがあれば .claude/upstream-backlog.jsonl に追記
   - セッションカウンタをインクリメント
   - 閾値に達したら「Issue を送りますか？」と提案
```

#### 4. /wd-migrate への統合

Phase 1（状態確認）に追加:

```
バックログ確認:
  .claude/upstream-backlog.jsonl が存在すれば件数を表示
  「N 件の未還元フィードバックがあります。migrate 前に確認しますか？」
```

#### 5. Issue 送信

ユーザーが承認したら:

```bash
# バックログから intent=issue のエントリを抽出
# Issue 本文を生成
gh issue create \
  --repo fruitriin/embodied-claude-wardrobe \
  --title "Downstream feedback: ..." \
  --body "$(cat <<'EOF'
## ダウンストリームからのフィードバック

環境: wardrobe-test
期間: YYYY-MM-DD 〜 YYYY-MM-DD
セッション数: N

### 提案

1. ...
2. ...

---
Generated by wd-contribution agent
EOF
)"
```

送信後:
- 送信したエントリをバックログから削除（または `intent` を `sent` に更新）
- セッションカウンタをリセット

### 保護対象への追加

以下をダウンストリーム固有ファイルとして `.gitignore` と `wd-migrate` の保護対象に追加:

- `.claude/upstream-backlog.jsonl`
- `.claude/upstream-session-counter`

## 実装順序

1. **バックログファイルの仕様確定** — JSONL スキーマ
2. **セッションカウンタ** — シャットダウン手順に組み込み
3. **wd-contribution の軽量モード** — `--quick` フラグ。差分ベースで分類する（後述）
4. **Issue 送信スクリプト** — `gh issue create` のラッパー
5. **/wd-migrate への統合** — Phase 1 にバックログ確認を追加

## 差分取得の戦略

`git diff` だけでは、こまめにコミットするユーザーの変更を検出できない。

### 案A: git diff（未コミット差分のみ）

```bash
git diff -- .claude/commands/ .claude/hooks/ memory-mcp/ ...
```

- 利点: 軽い、シンプル
- 欠点: コミット済みの改善を見落とす

### 案B: アップストリーム基点の差分（推奨）

`wardrobe-lock.json` のコミットハッシュを基点に、アップストリーム管理対象ファイルの全変更を見る:

```bash
# wardrobe-lock.json から前回のアップストリームコミットを取得
UPSTREAM_COMMIT=$(jq -r .commit .claude/wardrobe-lock.json)

# アップストリームを shallow clone
git clone --depth=1 https://github.com/fruitriin/embodied-claude-wardrobe.git /tmp/wd-upstream

# アップストリーム管理対象ファイルだけを比較
# ダウンストリームにあってアップストリームにない変更 = 還元候補
diff -rq /tmp/wd-upstream/.claude/commands/ .claude/commands/ --exclude="*.exp.md"
diff -rq /tmp/wd-upstream/memory-mcp/ memory-mcp/ --exclude="*.db" --exclude=".venv"
# ...
```

- 利点: コミット済みかどうかに関係なく、アップストリームとの真の差分が出る
- 欠点: ネットワークアクセスが必要（shallow clone）。/wd-migrate の Phase 3 と処理が重複する
- 最適化: /wd-migrate が clone したキャッシュを再利用できる

### 案C: ハイブリッド

- シャットダウン時の `--quick` は案A（軽い）
- /wd-migrate 時は案B（どうせ clone するので）
- 手動実行時はユーザーに選ばせる

**現時点の推奨: 案C**。日常は軽く、マイグレーション時に正確にやる
