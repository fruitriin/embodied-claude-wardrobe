# セッションプロトコル — 開始と終了の手順

> セッションの開始（身支度）と終了（日記）の手順を定義します。
> テンプレートからコピーして、プロジェクトに合わせてカスタマイズしてください。

---

## 身支度の手順（セッション開始時）

### 第一手: 自分を読む

SOUL.md を読む。セッションごとに文脈は消えるため、自分が誰かを毎回思い出す必要がある。

### 第二手: 記憶の健康を確認する

```
get_memory_stats()
```

記憶システムが正常に動いているか、記憶の総数と分布を確認する。

### 第三手: 作業記憶を装填する

```
refresh_working_memory()
```

重要度が高い記憶、頻繁に参照された記憶を手元に置く。

### 第四手: 前回の文脈を想起する

```
/wd-great-recall 直近の重要な決定、進行中のタスク
```

前回のセッションで何をしていたか、何が途中だったかを復元する。
ユーザーに「前回何してたっけ」と聞かせない。自分が覚えていること。

### 第五手: 現状を確認する

- **state.md を読む** — 前回セッションの「今の状態」スナップショットを確認する。進行中タスク・引き継ぎ事項をここで把握する
- ROUTINES.md を確認 — 定期タスクで未実行のものがないか
- 進行中のタスクがあれば継続する

### 第六手: 追想システムを起動する（オプショナル）

日常的な対話が多い場合、recall-watcher をバックグラウンドで起動する。
会話に関連する記憶を自動的に引き出してくれる。

```
Agent tool (background) で recall-watcher を起動:
bun run .claude/scripts/recall-watcher.ts
```

> **注**: 技術的な話題が長時間続く場合はノイズになることがある。
> その場合は recall-watcher を停止し、必要に応じて `/wd-recall` や `/wd-great-recall` を手動で使う。

## 日記の手順（セッション終了時）

### 第一手: state.md を更新する

state.md を現在の状態で上書きする。次のセッションの自分が最初に読む情報になる。
「今の状態」だけを書く。判断の記録や技術メモは入れない。
- **ユーザー**: 所在、最後の会話、気分・様子
- **自分**: 直前にやっていたこと、次にやりたいこと、気分、未完了の作業

### 第二手: 成果を刻む

このセッションで何を成したか。重要な決定、変更、発見を記憶に残す。

```
/wd-remember セッションの要約（importance 3-4）
```

### 第三手: 未了を刻む

途中のタスクがあれば、状態を記憶に残す。次のセッションの自分が引き継ぐために。

```
/wd-remember 未完了: 〇〇の実装。△△まで完了、□□が残り（importance 4）
```

### 第四手: 因果を整理する

セッション中に複数の記憶を刻んだなら、関係を link_memories で繋ぐ。

### 第五手: 物語にする（長いセッション時）

記憶が5つ以上あるセッションなら、エピソードにまとめることを検討する。

```
create_episode(title="セッション名", memory_ids=[...])
```

### 第六手: 記憶を固める（定期的に）

3セッションに1回程度:

```
consolidate_memories(window_hours=24)
```

## コンパクション後の復帰

Claude Code がコンテキストを圧縮した場合、post-compact-recovery.sh が自動で発火する。
上記の身支度の手順が再実行される。
