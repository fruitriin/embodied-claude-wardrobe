## サブタスク展開ルール

- **直近タスクだけ** 各ステップを展開する。遠いタスクは大きいままでいい
- 展開の粒度: **1セッションでチェックがつく** サイズ。終わった実感が得られる大きさ
- 次のタスクには「サブタスク展開」という準備タスクを直前に置く。いきなり飛び込まない
- 完了したサブタスクは [x] で残す。消さない。進捗が見える
- 完了サブタスクが増えて見通しが悪くなったら、`<details>` で折りたたむ:
  ```markdown
  <details><summary>完了したステップ (3/5)</summary>

  - [x] ステップ1
  - [x] ステップ2
  - [x] ステップ3

  </details>
  ```
- **候補が複数あって優先順位がつかないときは `/wd-dice` で選ぶ。迷う時間を減らす**
