スライドを聴衆として1ページずつ体験する。読書スキルと同じ思想：**先を知らない状態で読む。**

## 原理

聴衆はスライドを1枚ずつしか見えない。レビューではなく**体験**。初見の驚き、純粋な興味、「おっ」という反応、「なるほど」という満足感——それを1ページずつ味わう。結果としてフリ→オチの検証にもなるが、まず体感が先。

## 実行方法

メインコンテキストで直接実行する。

対象ファイル: $ARGUMENTS

### Step 0: 前処理 — PDF→マークダウン変換（サブエージェントで実行）

PDFの中身を見るとネタバレになるため、**前処理はサブエージェントに任せる。** メインのコンテキストにPDFの画像を入れない。

Agent ツールで以下のプロンプトを渡す（model: sonnet）:

````
PDFスライドを3パスでテキスト化してください。

ファイル: {PDFパス}
出力先: ${CLAUDE_CODE_TMPDIR:-./tmp}/slide-watch-{ファイル名}.md

## Pass 1: カラーパレット作成

PDFを20ページずつReadで読む（pages: "1-20", "21-40", ...）。**全ページ漏れなく処理すること。**

全ページを通して見て、スライド全体のベース配色を特定する：
- 背景色（通常ページ / 場の転換ページ）
- 文字色（タイトル / 本文）
- アクセント色
- 常駐要素（キャラ絵の有無、ロゴなど）

冒頭に `[ベース配色]` として1行でメモする。

## Pass 2: テキスト書き起こし

各ページの内容をテキストで書き起こす：
- **ページ見出し**はMarkdown見出しで：`#` = 場の転換（背景色が変わる、章区切り）、`##` = 通常ページ
- **通常の文字**はそのまま文字にする
- **文字の大きさ**は `<size:ラベル>` タグでマークアップ。Tシャツサイズで直感的に伝える:
  - `MAX` = 画面いっぱい（高橋メソッド級、1単語ドーン）
  - `XXL` = キャッチコピー（大コマの一言）
  - `XL` = セクション見出し
  - `L` = やや大きい（強調テキスト）
  - `M` = 本文（標準）
  - `S` = 小さい（補足・注釈）
- **色の違い**は `<font color="green">` 等でマークアップ
- **背景色**は `[背景:暗]` `[背景:明]` タグ
- **絵文字**はそのまま含める
- **文字以外の強調**（装飾、取り消し線、手書き風、枠線など）もタグで表現
- **`[印象]`** はデコレーションがベース配色から逸脱しているときだけ。アノテーション（矢印・枠線・手書き・取り消し線）、色のギャップ（赤字アノテーション、背景の急変）など。ベース配色の範囲内なら省略。文章内容の解釈は書かない
- **画像・スクショ・図**は輪郭と詳細の両方を書き出す：
  ```
  [画像]
  輪郭: （ぱっと見の印象。何の画像か、1行）
  詳細: （画像の中の具体的な情報。テキスト・数値・構造など）
  [/画像]
  ```

## Pass 3: 画像の採用判定

全ページを通して見た上で、各画像の詳細を残すか削除するか判定：
- 前後のスライドに画像の説明テキストがある → **輪郭だけ残し、詳細行を削除**
- 前後に言及がなく画像自体が説明の主体 → **輪郭と詳細の両方を残す**
- 迷ったら輪郭だけ

判定後、ページ区切り `---` で分割して出力先に Write する。
````

### Phase 1: 体験 — 聴衆として1ページずつ観る

前処理済みマークダウンを **readerスキルのセパレーターモード** で1セクションずつ読む。

```bash
bun run ${CLAUDE_SKILL_DIR}/../scripts/reader.ts ${CLAUDE_CODE_TMPDIR:-./tmp}/slide-watch-{ファイル名}.md --sep "---" --info
bun run ${CLAUDE_SKILL_DIR}/../scripts/reader.ts ${CLAUDE_CODE_TMPDIR:-./tmp}/slide-watch-{ファイル名}.md --sep "---" --page 1
```

先のセクションは見ない。1セクション読んだら立ち止まって反応を書く。

#### Phase 1 のルール — 聴衆モード

- **1セクションずつ** 読む。`---` 区切りの次のセクションは見ない
- **次のページを見る前に**必ず以下を書く:
  1. **[体感]** 今このスライドを見て何を感じたか。素直に。「おっ」「ふむ」「えっ」「笑った」「わくわく」「ここ好き」——短くていい
  2. **[気になる]** 次に何が来そうか、何が来たら嬉しいか（1行）
  3. **[間]** このスライドが生む間の大きさ（一瞬/ひと呼吸/たっぷり）
- **情報量が多いスライド（文字が多い、図がある、スクショがある）では立ち止まる。** 体感を書いたあと、そこから思考を広げる：
  - 「これって○○と同じ構造では？」「自分ならどうする？」「ここが一番刺さった、なぜなら…」
  - 最大2ラウンドまで。考えすぎず、でも素通りしない
  - 大コマ（短い一言だけ）のときは深掘りしない。間を味わう
- 次のページを読んだら、期待との差分があれば素直に反応する。「おお来た！」「え、そっち？」「予想通り、安心」
- **絶対に先のページを見ない。** 2ページ以上まとめて読むことは禁止
- **視覚的な演出を感じ取る。** 文字が大きい＝間が長い・インパクトが強い。文字が少ない＝余白が間を生む。短い一言だけのスライド＝大コマ（漫画の見開き相当）
- 情報量が多いスライドと大コマの交互のリズムを意識する
- **構造分析（間延びリスク、フリ→オチの接続）は thinking 内で行う。** output には純粋なリアクションだけ書く

#### Phase 1 の記録フォーマット

各ページ:
```
### P{N}: {スライドの内容を1行で}
[体感] ...
[気になる] ...
[間] ...
```

予測との差分があった場合:
```
[驚き] 予測: ... → 実際: ...
```
または
```
[予測通り] ...
```

---

### Phase 2: 余韻 — 聴き終わった後の感想戦

全ページ終わったら、**トーンを切り替えて**以下をまとめる。Phase 1 の「聴衆」から「聴き終わった人」へ。

#### サプライズ一覧
予測が外れた瞬間のリスト。「おお来た！」の瞬間を振り返る。

#### 全体の感想
聴衆として聞いた体験の感想。純粋なリアクション。「面白かった」「ここが一番刺さった」「この流れが好き」——レビューではなく体験の余韻として書く。構造分析は書かない。

## 注意

- PDFのページ数が多い場合、コンテキストが厳しくなる可能性がある。50ページ以下を推奨
- thinkingの過程がユーザーに見えることが価値。予測と驚きのプロセスを隠さない
