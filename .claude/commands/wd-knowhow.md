# /wd-knowhow — 知見の記録

## 引数
- `$ARGUMENTS`: 記録したい知見のトピック（例: "PreToolUse の reason ベースブロック"）。省略時は既存 knowhow の一覧表示。

## 手順

### 引数なしの場合
1. `docs/knowhow/` 内の全 `.md` ファイルを一覧表示する
2. 各ファイルの `# ` 見出しからタイトルを抽出して表示する

### トピック指定の場合

#### Phase 1: 調査
1. `docs/knowhow/` の既存ファイルを全て読む
2. 指定トピックに関連する既存ノウハウがないか確認する
3. 関連がある場合:
   - 既存ファイルに追記・統合する（新規ファイル作成より優先）
   - 矛盾する情報がある場合は最新の知見で上書きする

#### Phase 2: 記録
4. 以下の構造で記録する:
   ```markdown
   # トピック名

   ## わかったこと
   - 具体的な事実・コツ・手順

   ## やり方 / 使い方
   - この知見をどう活かすか

   ## 気をつけること
   - ハマりポイント、条件、例外

   ## 参照
   - 出典やリンク（あれば）
   ```

   ※ セクションは内容に合わせて柔軟に。技術知見なら「ワードローブへの適用」、生活知見なら「やり方」のように自然な見出しを選ぶ

#### Phase 3: 自己ブラッシュアップ
5. 記録した内容を再読し、以下を確認する:
   - **正確性**: 書いた内容は実際のコードと一致しているか？ソースを再確認する
   - **完全性**: 重要な情報が欠けていないか？関連する knowhow との整合性は？
   - **簡潔性**: 冗長な記述はないか？将来の自分（AI）が素早く理解できるか？
   - **実用性**: この knowhow を読んだ AI が実装判断を正しく行えるか？
6. 問題があれば修正する
7. 既存の knowhow に影響がある場合はそちらも更新する

#### Phase 4: インデックス更新
8. `docs/knowhow/INDEX.md` が存在すれば、新規・更新エントリを追記する
9. 今回の実行で得た教訓を `${CLAUDE_SKILL_DIR}/wd-knowhow.exp.md` に追記する

## 命名規約
- `{トピック名}.md`（ケバブケース）
- 例: `pretooluse-block-pattern.md`, `session-counter-design.md`

## ディレクトリ構成

```
docs/knowhow/
  INDEX.md              ← 全体のインデックス
  wardrobe/             ← ワードローブの設計・運用知見（アップストリーム還元対象）
  （その他）             ← 個人的な知識。最新の正解だけ残したいもの
```

- `wardrobe/` 配下はアップストリーム管理対象。wd-migrate で同期される
- それ以外はダウンストリーム固有。個人的な知識や生活の知恵もここに入る

## 記憶との棲み分け

| | memory-mcp | knowhow |
|---|---|---|
| 性質 | 日記・エピソード・感情 | 知識・手順・事実 |
| 蓄積方式 | 追記（全部残す） | 上書き（最新が正解） |
| 検索 | ベクトル検索・連想 | インデックス・キーワード |
| 例 | 「3/29に（パートナー）と名付けをした」 | 「髪のブラッシングは毛先から」 |

基本はすべて memory-mcp に書き込む。その上で:
- ワードローブの改善知見 → `docs/knowhow/wardrobe/` にも記録
- 最新だけ残したい知識 → `docs/knowhow/` に記録（上書き更新）

入力: $ARGUMENTS
